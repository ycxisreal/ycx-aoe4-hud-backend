# AoE4 HUD 后端接口文档

本文档描述后端 WebSocket 接口协议与字段说明，供前端对接使用。

## 基本信息

- 协议：WebSocket
- 地址：`ws://127.0.0.1:8765`
- 消息格式：JSON
- 字段说明：所有消息均包含 `type`、`version`、`ts`、`payload`

## 通用消息结构

```json
{
  "type": "MESSAGE_TYPE",
  "version": 1,
  "ts": 1700000000000,
  "payload": {}
}
```

字段说明：
- `type`：消息类型
- `version`：协议版本（固定 1）
- `ts`：毫秒级时间戳
- `payload`：消息体

## 前端 -> 后端

### CONFIG_SET

用于下发屏幕信息、ROI 列表、识别频率与模板集配置。

```json
{
  "type": "CONFIG_SET",
  "version": 1,
  "ts": 1700000000000,
  "payload": {
    "clientId": "client-001",
    "screen": { "width": 2560, "height": 1440, "dpiScale": 1.25, "displayId": 0 },
    "rois": [
      {
        "id": "timer",
        "name": "Timer",
        "rect": { "x": 100, "y": 50, "w": 120, "h": 40 },
        "kind": "timer",
        "expected": { "charset": "digits_colon", "maxLen": 5 }
      }
    ],
    "recognition": { "enabled": true, "hz": 2 },
    "tts": { "enabled": true, "voice": "zh-CN-XiaoxiaoNeural", "rate": "+0%", "volume": "+0%" }
  }
}
```

字段说明：
- `screen.displayId`：显示器索引，默认 `0`
- `screen.dpiScale`：系统缩放比例（例如 125% 传 `1.25`）。若不传，后端会尝试读取系统缩放并按比例修正 ROI。
- `rois[].kind`：可选值：
  - `timer`
  - `idle`
  - `res_food` / `res_wood` / `res_gold` / `res_stone`（可扩展更多 `res_*`）
  - `gather_food` / `gather_wood` / `gather_gold` / `gather_stone`（可扩展更多 `gather_*`）
  - `custom`（保留）
- `recognition.hz`：识别频率，建议 `1~2`
- `tts.voice`：edge-tts 音色，默认 `zh-CN-XiaoxiaoNeural`
- `tts.rate`：语速百分比字符串（例如 `+0%`、`+20%`、`-10%`）。兼容旧整数语速并自动换算。
- `tts.volume`：音量百分比字符串（例如 `+0%`、`-20%`）。兼容旧 `0~1` 浮点值并自动换算。

### START

启动识别循环。

```json
{ "type": "START", "version": 1, "ts": 1700000000000, "payload": {} }
```

兼容别名：
- `RECOG_START`
- `OCR_START`

### STOP

停止识别循环。

```json
{ "type": "STOP", "version": 1, "ts": 1700000000000, "payload": {} }
```

兼容别名：
- `RECOG_STOP`
- `OCR_STOP`

### PING

```json
{ "type": "PING", "version": 1, "ts": 1700000000000, "payload": { "seq": 1 } }
```

## 后端 -> 前端

### BACKEND_STATUS

```json
{
  "type": "BACKEND_STATUS",
  "version": 1,
  "ts": 1700000000000,
  "payload": {
    "state": "ready",
    "message": "optional_message",
    "details": {}
  }
}
```

`state` 可选值：
- `starting`
- `ready`
- `running`
- `stopped`
- `error`

### DATA

```json
{
  "type": "DATA",
  "version": 1,
  "ts": 1700000000000,
  "payload": {
    "fields": {
      "timer": { "value": "12:34", "conf": 0.8 },
      "idleVillagers": { "value": 2, "conf": 0.7 },
      "resources": {
        "food": { "value": 120, "conf": 0.9 },
        "wood": { "value": 200, "conf": 0.9 }
      },
      "gatherers": {
        "food": { "value": 6, "conf": 0.8 }
      }
    },
    "frameTs": 1700000000000,
    "quality": { "ok": true, "reason": null }
  }
}
```

字段说明：
- `conf`：置信度（0~1，数值越大越可靠）
- `quality.ok`：质量是否可靠
- `quality.reason`：不可靠原因（`capture_failed` / `timer_invalid` / `timer_regression` / `value_negative` / `value_jump` 等）

### ALERT_EVENT

```json
{
  "type": "ALERT_EVENT",
  "version": 1,
  "ts": 1700000000000,
  "payload": {
    "id": "idle_villager",
    "level": "warn",
    "text": "有闲置村民",
    "spoken": true,
    "cooldownMs": 5000
  }
}
```

字段说明：
- `level`：`info` / `warn` / `critical`
- `spoken`：是否已触发语音播放
- 当规则条件持续成立时，后端会按 `cooldownMs` 周期重复发送 `ALERT_EVENT`

### PONG

```json
{ "type": "PONG", "version": 1, "ts": 1700000000000, "payload": { "seq": 1 } }
```

## 错误与异常说明

- `templates_missing`：模板集加载失败
- `config_invalid`：配置解析失败
- `config_missing`：未配置就尝试 START

后端尽量不主动断开连接，错误情况下会通过 `BACKEND_STATUS` 或 `quality` 反馈问题。

## OCR 说明

- 后端当前使用 Tesseract OCR 识别数字与计时器。
- 不再使用模板集与模板文件。
- 需要在系统中安装 Tesseract 引擎，并确保 `tesseract` 在 PATH 中可访问。

## TTS 说明

- 后端当前使用 `edge-tts` 进行语音合成。
- `edge-tts` 依赖网络连接到微软语音服务，离线状态下无法播报。

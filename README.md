# AoE4 HUD 后端

这是一个aoe4HUD依赖的本地提醒后端。  
前端：https://github.com/ycxisreal/ycx-aoe4-hud-frontend  
启动后端并连接前端配置后，会自动读取 HUD 信息，按规则进行语音提醒，帮助你更快发现“闲置村民、资源配比失衡、关键时间点”等局内问题。

## 这个项目可以用来干嘛

- 实时读取对局 HUD 数据：计时器、闲置村民、资源数、采集村民分配
- 自动语音提醒关键问题：例如闲置村民、资源偏高/偏少、采集占比过高
- 支持时间窗与节奏提醒：例如 `12:00`、`15:00` 等时间节点进行一次性提醒
- 识别结果实时推送前端：前端可展示数据状态、提醒事件、识别质量  
- 部分规则配置示例如下：  
>**规则：闲置村民播报**  
触发条件（同时满足）：  
1）计时器有效；  
2）idleV ≥ 1。  
holdFor：2 秒。  
播报内容：“有闲置村民”。  
冷却：15 秒。  
累加冷却：不启用。  
重置条件：idleV == 0 时，视为不满足并清空任何内部状态。  

>**规则：09:00-15:00 木头资源偏高**  
适用时间：09:00-15:00。  
触发条件（同时满足）：  
1）W > 1000；  
2）W > max(F, G) * 1.6。  
holdFor：12 秒。  
播报内容：“木头资源偏高”。  
冷却：45 秒。  
累加冷却：+15 秒。  
重置条件：任一触发条件不满足时（例如 W ≤ 1000 或 W ≤ max(F,G)*1.6），清零累加。 

## 未实现的功能  
本项目为自用项目，并未实现含额外资源的民族的数据识别与规则配置，如：拜占庭与马其顿等。以及没有实现可视化配置规则功能，添加规则需自行在代码中添加。
## 主要能力

- 自动抓屏 + ROI 识别（Windows）
- 识别结果稳定化与合法性校验
- 规则引擎（支持 `holdFor`、冷却、累加冷却、时间窗、一次性触发）
- 语音播报（`edge-tts`）
- WebSocket 本地通信

## 运行环境

- Windows 10/11
- Python 3.10 及以上
- 需要在本地安装 Tesseract OCR
- 安装完毕可以自检：
```powershell
tesseract --version
```
- 可联网（`edge-tts` 需要在线语音服务）


## 安装依赖

在项目根目录执行：

```powershell
pip install -r requirements.txt
```

## 启动

在项目根目录执行：

```powershell
python main.py
```

默认 WebSocket 地址：
- `ws://127.0.0.1:8765`

## 快速使用步骤

1. 启动后端（`python main.py`）
2. 前端连接 WebSocket
3. 发送 `CONFIG_SET`（屏幕信息、ROI、识别频率、TTS）
4. 发送 `START` 开始识别
5. 运行过程中接收 `DATA` 和 `ALERT_EVENT`
6. 需要停止时发送 `STOP`

## 常见问题

1. 没有识别数据
- 检查前端是否以及完成区域标定，如果未标定区域则无法进行识别
- 检查外部依赖tesseract OCR是否安装并加入系统 `PATH`路径  

2. 有提醒事件但没有语音
- 检查网络是否可用（`edge-tts` 依赖联网）
